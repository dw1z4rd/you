# You.

A psychometric instrument disguised as a cosmogony.

An LLM asks questions entailed by your previous answers — not pre-designed, dynamically generated. The universe that emerges is a map of your mind. Two players build fundamentally different universes because they *are* fundamentally different.

---

## How It Works

**The void asks one question:** *Does the void notice itself?*

Answer no — and nothing emerges. The void remains undifferentiated. This too is a universe.

Answer yes — and a first distinction is made: the noticing and the noticed. One becomes two. Every possible world lies downstream.

From there, each answer is translated into a metaphysical axiom. Each axiom constrains what questions can follow. After 14 axioms, your universe crystallizes — a narrative and a structured schema that drives procedural 3D world generation. You then enter that world and can invite others to visit.

**Two layers of self-mapping:**
1. **Question layer** — conscious, philosophical. What you believe about consciousness, causality, time, identity, matter, meaning.
2. **Interaction layer** — behavioral, embodied. How you move through the world, where you look, what you do. *(future)*

## Features

### Finished

**Question engine**
- First question hardcoded: *"Does the void notice itself?"* — a no answer produces the void ending, a valid universe with nothing in it
- All subsequent questions dynamically generated by an LLM, each logically entailed by the axioms established so far
- Questions address fundamental structure of existence: consciousness, causality, time, identity, matter, multiplicity, meaning, will, finitude, relation
- Forced-choice format — yes/no or binary ontological positions, never open-ended
- Domains tracked to avoid revisiting covered ground
- JSON-structured response with question, domain, and exact choice labels

**Axiom derivation**
- Each answer translated into a precise present-tense metaphysical truth
- Coherence-checked against prior axioms in the prompt
- Displayed to the player before the next question appears

**Universe synthesis**
- At 14 axioms, the LLM synthesises a narrative describing the universe that emerged
- Simultaneously produces a structured `UniverseSchema` — 13 typed enum fields covering metaphysical and visual dimensions (consciousness, identity, time, causality, meaning, matter, life, palette, density, scale, geometry, light, entropy)
- Schema drives all downstream 3D world generation parameters

**Void aesthetic UI**
- Pure black, centered, no chrome
- Typewriter animation on each question
- Fly-in / fade-out transitions between questions
- Axiom revealed for 3 seconds before advancing
- Togglable axiom log during the session
- Void ending screen for the null universe

**3D world generation (scaffolded)**
- `UniverseSchema` → `WorldParams` mapping fully defined: palette → hex colors, scale → terrain size and fog distances, light → sun intensity and shadow casting, geometry → terrain shape, entropy → max height and roughness, consciousness/matter/life → gaze reactivity, matter solidity, and movement
- Three.js scene with `WebGLRenderer`, `PerspectiveCamera`, `PCFSoftShadowMap`, ACES filmic tone mapping
- Ambient + directional lighting, configurable by schema
- Particle system (fog, snow, bioluminescent spores) driven by schema
- Terrain: `PlaneGeometry` with working placeholder height functions (rolling, jagged, wavy, flat) per schema geometry field — real noise pending
- First-person player controller: pointer lock, WASD movement, mouse look
- Orbs rendering for other players in the scene

**Multiplayer**
- PartyKit (Cloudflare Durable Objects) for real-time sync and persistence
- Owner creates a room when entering their world — room ID and owner secret embedded in URL
- Visitor URL: `/world?room=<ID>` — no secret needed to visit
- Position sync at ~80ms intervals, other players visible as glowing orbs
- Permission system: visitors can request edit access, owner grants or revokes per-connection
- Room state (universe schema, synthesis, permissions) persisted in PartyKit storage — survives owner disconnect
- "Copy invite link" button in the world view

**Admin panel**
- Protected route at `/admin` — password login via `ADMIN_TOKEN` env var, 8-hour session cookie
- Overview: total universes created, active rooms, players currently online
- Schema distributions: for each of the 13 `UniverseSchema` fields, a text bar chart showing percentage breakdown across all universes ever created
- Room list: room ID, key schema values, active player count, created and last-active timestamps
- Data sourced from a separate `registry` PartyKit party that universe rooms notify on join/leave

## Getting Started

### Prerequisites

- Node.js 20+
- [Ollama](https://ollama.com) running locally with at least one model pulled (recommended: `llama3.1:8b` or larger)

### Install

```powershell
npm install
```

### Configure

```powershell
copy .env.example .env
```

Edit `.env`:

```env
OLLAMA_MODEL=llama3.1:8b          # or any model you have pulled
OLLAMA_BASE_URL=http://localhost:11434

ADMIN_TOKEN=your-secret-password  # for /admin

PARTYKIT_HOST=localhost:1999
PUBLIC_PARTYKIT_HOST=localhost:1999
```

### Run

Open two terminals:

```powershell
# Terminal 1 — SvelteKit dev server
npm run dev

# Terminal 2 — PartyKit (multiplayer + registry)
npm run party
```

Navigate to `http://localhost:5173`.

## Multiplayer

When a player finishes their universe and clicks **enter**, they are assigned a room code and taken to `/world?room=<ID>&secret=<secret>`. Sharing the URL without the `?secret=` parameter gives visitors read-only access by default.

**Owner controls:**
- See connected visitors in the bottom-right panel
- Grant or revoke edit permissions per connection
- Visitors with edit access can answer questions that evolve the shared universe *(question-answering in-world coming soon)*

**Visitors:**
- Click **request edit access** to prompt the owner
- Can always move freely through the world regardless of permission status

---

## Admin Panel

Visit `/admin`. Login with the value of `ADMIN_TOKEN` from your `.env`.

The panel reads from the PartyKit registry — if PartyKit is not running, the panel loads with empty stats rather than erroring.

## Planned Features

**Terrain generation**
- Replace placeholder sin/cos height functions with proper simplex noise (rolling, jagged, wavy shapes)
- Animated wavy terrain driven by `time` axiom
- Terrain raycasting for accurate ground-following camera

**World objects**
- Procedurally placed objects driven by `objectDensity`, `objectScale`, and `objectShape` schema fields
- Organic shapes (trees, growths), crystalline formations, geometric primitives
- Bioluminescent glow on objects when `light === 'bioluminescent'`

**Matter and life effects**
- Ghostly wireframe rendering for `matter === 'illusory'` universes
- Translucent terrain for `matter === 'emergent'`
- Ambient movement animations for `life === 'active'` and `life === 'subtle'`

**Gaze-reactive world**
- When `consciousness === 'substrate'`, terrain and objects respond to where the player looks
- Pointer-lock gaze direction drives ripple effects in the landscape

**Day/night cycle**
- When `time === 'cyclical'`, a full day/night cycle runs at `timeSpeed` rate
- Dynamic sky color, sun arc, and ambient light shift across the cycle
- When `time === 'absent'`, the world is frozen in a single light state

**In-world question answering**
- After entering the world, questions can continue to appear and reshape it
- Permitted visitors can submit answers that evolve the shared universe in real time
- World geometry reacts to new axioms without a page reload

**Interaction layer**
- Track behavioral signals: how long a player lingers in certain areas, movement patterns, where they look
- Feed these signals back into the LLM as a second layer of self-mapping alongside explicit answers

**Universe browser**
- Public gallery of universes (opt-in)
- Filter by schema field values — find all universes where `consciousness === 'absent'` or `time === 'cyclical'`
- Visit any public universe as a guest without a room invite

**Cloud LLM support**
- Already architected: swap `createOllamaProvider` for `createAnthropicProvider` or any other `LLMProvider`
- Synthesis quality improves significantly with larger models (recommended: Claude 3.5+ or DeepSeek V3)

**Mobile**
- Touch-based movement controls (virtual joystick)
- Gyroscope look on supported devices

**Export**
- Download your universe narrative as a PDF
- Share a static snapshot of your schema and axiom log
